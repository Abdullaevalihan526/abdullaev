Название программы: Weather Diary (Дневник погоды)

Краткое описание
Weather Diary — это десктопное приложение для учёта и анализа погодных условий. 
Программа позволяет:
добавлять записи о погоде с указанием даты, температуры, описания и наличия осадков;
просматривать все записи в табличном виде;
фильтровать записи по дате и температуре;
сохранять данные в формате JSON и загружать их из файла.

Приложение разработано на языке Python с использованием стандартной библиотеки tkinter для графического интерфейса. 
Данные хранятся в локальном JSON‑файле.

Как запустить приложение
Требования:Python версии 3.6 или выше.

Порядок запуска:

Клонируйте репозиторий на свой компьютер:

bash
git clone <ссылка_на_репозиторий>
cd WeatherDiary
Убедитесь, что у вас установлен Python. Проверить версию можно командой:

bash
python --version
Запустите приложение:

bash
python WeatherDiary.py
После запуска откроется окно приложения с полями для ввода данных и таблицей для отображения записей.

Примеры использования
Пример 1. Добавление новой записи
В поле «Дата (YYYY‑MM‑DD)» введите дату в формате ГГГГ-ММ-ДД, например: 2024-05-15.
В поле «Температура (°C)» укажите температуру числом, например: 22.5.
В поле «Описание погоды» напишите краткое описание, например: Солнечно, лёгкий ветер.
Если в этот день были осадки, установите флажок «Осадки».
Нажмите кнопку «Добавить запись». Новая запись появится в таблице внизу окна.

Пример 2. Фильтрация записей
По дате:
В поле «Фильтр по дате» введите нужную дату, например: 2024-05-10.
Нажмите кнопку «Применить фильтры». В таблице отобразятся только записи за указанную дату.
По температуре:
В поле «Фильтр по температуре (>) введите пороговое значение, например: 15.
Нажмите «Применить фильтры». В таблице останутся записи, где температура выше 15 °C.

Пример 3. Сохранение и загрузка данных
После добавления нескольких записей нажмите кнопку «Сохранить в JSON». Данные сохранятся в файл weather_diary.json в папке проекта.
Закройте и снова запустите приложение.
Нажмите кнопку «Загрузить из JSON», чтобы восстановить ранее сохранённые записи в таблице.
Возможные тесты для проверки работы
Тест корректности ввода:
Попробуйте добавить запись с пустой датой — должно появиться сообщение об ошибке.
Введите в поле температуры текст вместо числа — должно появиться сообщение об ошибке.
Оставьте поле описания пустым — должно появиться сообщение об ошибке.
Тест фильтрации:
Добавьте несколько записей с разными датами и температурами.
Примените фильтр по дате — убедитесь, что отображаются только записи за выбранную дату.
Примените фильтр по температуре — убедитесь, что отображаются записи с температурой выше указанного значения.
Тест сохранения/загрузки:
Добавьте 2–3 записи.
Сохраните данные в JSON.
Закройте приложение, откройте заново.
Загрузите данные из JSON и проверьте, что все записи корректно отобразились в таблице.
